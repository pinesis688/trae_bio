# 重写题库生成程序计划

## 概述

重写 `server.py` 中的 AI 题目生成逻辑，使用 >390B 大模型，实现 1-3 分钟/题的高质量生成流程。题目分为高考题和竞赛题，命题规则参考高考/竞赛大纲，并融入《高中必刷题（生物学）》和奥赛讲义配套习题册的命题风格。保留 HTTP 服务、Supabase 同步、知识图谱对齐等基础设施不变。

## 当前状态分析

### 现有模型配置（server.py 第 44-46 行）
- `FAST_MODEL = "meta/llama-3.1-70b-instruct"` — 70B，**不满足 >390B 要求**
- `SELF_CHECK_MODEL = "meta/llama-3.3-70b-instruct"` — 70B，**不满足**
- `QWEN = "qwen/qwen3.5-397b-a17b"` — 397B，已定义但**未作为默认生成模型**
- 高考模式进一步降级到 `meta/llama-3.1-8b-instruct` — 8B，**完全不满足**

### 现有生成流程问题
1. **批量生成**：batch_size=3-8，一次 API 调用生成多题，质量稀释
2. **自检模型太小**：8B/70B 自检，无法可靠验证 397B+ 模型的输出
3. **提示词单薄**：仅 5-6 条通用规则，未参考具体教材/习题册风格
4. **无命题规则**：缺少高考/竞赛的具体命题规范（情境创设、干扰项设计、认知层次）
5. **速度优先**：当前设计为秒级/题，与用户要求的 1-3 分钟/题矛盾

## 方案设计

### 模型选择
- **主生成模型**：`deepseek-ai/deepseek-v4-pro`（671B MoE，推理最强，中文顶级）
- **自检模型**：`qwen/qwen3.5-397b-a17b`（397B，用不同模型交叉验证，避免同源偏差）
- **备用模型**：`z-ai/glm-5.1`（当主模型限流时自动切换）

### 生成流程（1-3 分钟/题）
```
┌─────────────┐    ┌──────────────┐    ┌──────────────┐    ┌────────────┐
│ 1. 命题生成  │───▶│ 2. 答案自检  │───▶│ 3. 质量审核  │───▶│ 4. 入库    │
│ deepseek-v4  │    │ qwen-397b    │    │ 规则+启发式   │    │ pool+SB    │
│ pro (671B)   │    │ 交叉验证     │    │              │    │            │
│ ~30-60s      │    │ ~30-60s      │    │ ~5s          │    │ ~2s        │
└─────────────┘    └──────────────┘    └──────────────┘    └────────────┘
```

1. **命题生成**（~30-60s）：单题生成（batch_size=1），使用增强提示词（含命题规则+教材风格），要求模型输出题目+答案+详细解析+命题意图
2. **答案自检**（~30-60s）：用 Qwen-397B 交叉验证答案（温度 0.1 和 0.5 各一次），两次均与标答一致才通过；不一致则触发一次修正重试
3. **质量审核**（~5s）：科学事实校验 + 干扰项质量 + 去重 + 解析完整度 + 命题规则合规
4. **入库**（~2s）：写入 pool.json + Supabase

### 增强提示词设计

#### 高考题系统提示词要点
- 参照《普通高中生物学课程标准（2017年版2020年修订）》命题
- 参考《高中必刷题（生物学）》风格：情境真实、选项精细、解析逐条
- 命题规则：
  - 情境创设：使用教材经典实验、生活实例、科研情境
  - 认知层次：理解→应用→分析，禁止纯识记题
  - 干扰项设计：基于学生常见 misconceptions，非随机编造
  - 选项规范：4个选项同层级、同长度范围、互斥
  - 解析要求：逐选项说明对错原因，≥100字，含知识点回归
- 禁止：模糊表述（"某种""某生物"）、"以上都对/都错"、超纲术语

#### 竞赛题系统提示词要点
- 参照全国中学生生物学联赛大纲（CBO大纲）命题
- 参考奥赛讲义配套习题册风格：考察深度理解、实验设计、数据分析
- 命题规则：
  - 知识范围：覆盖大学普通生物学，可涉及具体酶、信号通路、实验技术
  - 情境创设：科研前沿、经典实验复现、数据图表分析
  - 干扰项设计：体现竞赛生高级误解，需深入理解才能排除
  - 图表题：支持生成 Markdown 表格数据题
  - 解析要求：逐选项分析 + 知识点延伸，≥120字

## 假设与决策
- 假设 NVIDIA NIM 免费 API 对 `deepseek-ai/deepseek-v4-pro` 的调用不限量（当前 key 已在使用）
- 假设 671B 模型单次生成 + 397B 自检总耗时约 1-3 分钟（取决于 API 响应速度）
- 决定使用双模型交叉验证（DeepSeek 生成 + Qwen 自检）而非同模型自检，以提高错误检出率
- 决定保留 auto_refill 但降低默认触发水位（POOL_MIN=200→100），避免大模型自动补货过慢
- 决定不修改 scripts/ 目录下的辅助脚本（batch_gen.py 等通过 import server 自动继承新逻辑）
