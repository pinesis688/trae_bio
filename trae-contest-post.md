## 0. 打个招呼

我是一名高一学生，马上要去市里组织的生物竞赛集训营（7 月 1 号到 12 号），去了之后基本就没时间写代码了，所以赶着期末这几周把这个网站做出来了。

做这个东西的起因很简单：备赛的时候找真题太费劲，论坛、QQ群、网盘到处翻，解析经常是模糊的照片或者干脆没有。身边几个同学生物底子其实不错，但就是没人带着系统练，光靠看书刷题效率低。我就想，能不能做个网站把这些东西整合起来，免费给大家用。

写代码的基础说起来有点好笑——小学六年级的时候在记事本里写过几行 HTML，拼过几个 `<table>` 和 `<marquee>` 跑马灯，仅此而已。上高中之后基本没碰过代码，这个项目是我第一次正经写 JavaScript。整个网站基本上是跟 TRAE 对话做出来的。最开始我连 SPA 是什么都不知道，就跟它说"我想做个单页网站，点导航不要刷新页面"，它就给我写了个 hash 路由；我说"AI 回答能不能像 ChatGPT 那样一个字一个字蹦出来"，它就告诉我用 SSE + ReadableStream；六家大模型的接口格式不一样我自己肯定要翻好几天文档，TRAE 直接帮我封装了个统一的 `ai-client.js`，流式输出、错误重试、超时处理都弄好了。虚拟实验室的 Canvas 绘制也是它带着我从零做的，从画一个显微镜开始到能模拟调焦看细胞，改了不知道多少轮。

说真的，以我那点 HTML 皮毛，没有 TRAE 这个项目根本做不完。

## 1. Demo 简介

**是什么**：一个纯前端的生物学习网站，浏览器打开就能用，不用装 App，不用注册也能刷题，零服务器成本。

**面向谁**：高中生物竞赛生（全国中学生生物学联赛/CNBO）、高考生物想冲高分的同学、以及任何对生物感兴趣但找不到系统免费资源的自学者。

**主要功能**：

- **AI 生物导师**：支持 DeepSeek、智谱 GLM、阿里通义、月之暗面 Kimi、NVIDIA NIM、硅基流动六家大模型，自己在设置里填 API Key 就行（Key 存在你浏览器里，我看不到）。三个导师人格可以切——通用导师什么都能聊，孟德尔专讲遗传学，达尔文专讲进化论。回答是流式输出的，打字机效果，支持 Markdown 和公式。每天限 100 次，防止烧 Key。

![AI导师](test_screenshots/tutor.png)

- **专项练习 + 72 题全真模考**：题库目前 330+ 道题，按四大模块分类（生化分子细胞 / 植物微生物 / 动物生态 / 遗传演化），支持高考/竞赛两种模式、基础/进阶/挑战三档难度、基础知识/逻辑推理两种题型。做完自动判分，错题自动收入错题本，可以按知识点重做。还有 72 题 150 分钟的全真模考，对齐 2025 年联赛新赛制。

![专项练习](test_screenshots/practice.png)

- **交互式知识图谱**：40 多个核心概念按力导向布局排列，八个学科用不同颜色区分（紫色=细胞、橙色=遗传、绿色=生态……），节点大小反映知识点的联系紧密程度，颜色深浅反映你对这个知识点的掌握程度（绿色掌握、黄色一般、红色薄弱）。点哪个节点就直接跳去练哪个知识点的题，算是"哪里不会点哪里"。

![知识图谱](test_screenshots/knowledge-graph.png)

- **虚拟生物实验室**：显微镜观察、叶绿体色素提取分离、DNA 粗提取、微生物培养这几个高频考点实验，在网页里模拟操作。要按正确步骤选工具做实验，选错了会提示你哪里不对，连续错两次还能点"为什么这样做"看原理。比光看课本上的实验步骤图直观多了。

![虚拟实验室](test_screenshots/bio-lab.png)

- **错题本 + 知识卡片 + 学习仪表盘**：错题本能按知识点和错误原因分类，支持深度复盘；知识卡片用 FSRS 间隔重复算法安排复习，不用自己想什么时候该复习什么；仪表盘有每日目标、连续打卡、正确率统计、学习趋势，还有番茄钟和 AI 目标拆解（输入"两周内学完遗传"它帮你拆成每天的任务）。

![学习仪表盘](test_screenshots/dashboard.png)

- 其他还有：生物学史时间轴（从林奈到 AlphaFold）、智能诊断（答够 10 题后 AI 分析你的薄弱环节）、社区讨论区和问答悬赏、联赛倒计时、每日一题、教师模式（老师可以发 8 位用户码加学生）、全局排行榜。

设计上用了鼠尾草绿当主色，赤陶橙做暖色点缀，字体是霞鹜文楷（本地加载，不走 CDN），支持深浅主题切换，手机电脑都适配。

![首页](test_screenshots/home.png)

## 2. Demo 创作思路

**灵感来源**：就是自己备赛的真实痛点。准备联赛这段时间，真题要从论坛、QQ群、网盘到处搜，好的解析要么是几百块的教辅书要么是培训机构的付费课。联赛去年改了新题型（72 道多重判断题、150 分钟），市面上连个对齐新题型的免费刷题工具都找不到。

**想解决的问题**：
- 竞赛资源太贵太散，三四线城市和普通学校的学生很难系统训练；
- 课本上的实验只能看图片，很多学校根本没条件做实验，学生对实验流程没概念；
- 刷了很多题但不知道自己哪里薄弱，盲目刷题效率低；
- 现有学习 App 要么收费、要么要装客户端、要么要把数据上传给厂商。

**为什么这么做（技术取舍）**：
- 纯前端不是因为不会写后端，是因为我一个高中生租不起也维护不起服务器。Supabase 免费版存数据够用，AI 让用户自己带 Key，部署到 Vercel 上一分钱不花，多少人用都没事，也不存在"作者服务器挂了大家都用不了"的问题；
- AI 不是拿来帮你做题的，是拿来帮你诊断哪里不会的。知识图谱看颜色就知道哪个知识点薄弱，导师帮你讲清楚为什么错，关键还是得自己练；
- 虚拟实验室用原生 Canvas 画，没引入重型图形库，加载快、手机上也能跑；
- 题库从 330 题起步，不贪多，先保证每道题都有解析、都对齐考点。

## 4. TRAE 实践过程

全程用 TRAE IDE 开发，挑几个有截图的真实任务说说。

**任务一：批量修 bug + 清理动画残留内容**

有一次我攒了一堆问题一起扔给 TRAE：拍照录题的 OCR 错误率太高（让它找支持中英文斜体识别的 OCR 方案）、仪表盘数据有误、成就章要加图标、学习热力图、讨论区里 AI 以科学家名字自称不合适要改成学科智能体、还有——AI 导师回答里老是漏出 `[[ANIM:speciation]]` 这种内部标记和没渲染的 `<svg>` 源码（就是之前删动画模块时没清干净），要求把动画相关内容全删掉、SVG 必须实时渲染。TRAE 收到后自己建了 todo 列表，把这些任务拆成五项逐个修，同时加载了 `discussion.js`、`tutor.js`、`ai-client.js`、`photo-quiz.js`、`wrongbook.js` 等十个文件作为上下文，一次性改完。截图里能看到它的回复还在输出 SVG 源码——这是修之前的状态，改完后这些标记就被正则清洗掉了。

![开发截图1：批量修bug与动画内容清理](test_screenshots/dev-01-bugfix-anim.png)

**任务二：修复 server.py 题目生成写入 Supabase 崩溃**

这是 6 月 19 号的任务"修复server.py题目生成问题"，耗时 9 分 34 秒。当时 AI 生成题目写入 Supabase 一直报错，TRAE 排查后发现两个问题：一是 `load()` 函数里 Supabase 返回的题目 `id` 可能是 `None`，合并数据时直接 `KeyError: 'id'` 崩溃；二是 Supabase 的 RLS（行级安全策略）拦住了写入——server.py 用的是 anon key，但 questions 表的 INSERT 策略只允许 admin 操作。TRAE 给出了修复方案：代码层面改成防御性取值加自动生成 id；数据库层面给出了需要在 SQL Editor 里执行的 SQL，包括补 weight/fb_good/fb_bad/target 四个字段、创建 `questions_service_insert` 和 `questions_service_update` 策略让 server.py 能匿名写入指定来源的数据。改完后它自己跑了一遍验证，成功生成了 2 道题。

![开发截图2：修复server.py写入Supabase崩溃](test_screenshots/dev-02-serverpy-fix.png)

**任务三：加开源协议 + 重写 README**

6 月 20 号的任务"增加开源协议"，耗时 3 分 16 秒。我跟它说"README 写详细点，明确用了什么算法、优势是什么，把许可证一起传到 main 分支"。它自己分析了项目结构，然后一口气做了这些事：完整重写了 README.md（新增约 430 行），把七大核心算法都列出来说明（大模型生成用 Qwen3.5-397B-a17b 加 JSON Mode 强约束、双模型 Qwen+GLM-4.7 交叉验证"双盲同行评审"通过率约 70%、四层格式校验、加权题目池采样等等），新增 DEPLOYMENT.md 部署文档和 PERFORMANCE_OPTIMIZATION.md 性能优化说明，添加 LICENSE 文件（CC BY-NC-SA 4.0），最后自动 commit 并推送到 GitHub main 分支，还创建了 PR 分支 `trae/solo-agent-jJUelT`。

![开发截图3：增加开源协议与重写README](test_screenshots/dev-03-license-readme.png)

这三个任务只是其中几个典型的，其他还有从零搭 SPA 路由、封装六家大模型的 ai-client.js、Canvas 虚拟实验室、SQL 初始化脚本的坑、字体精简等等，基本都是这样——我用自然语言描述要做什么，TRAE 拆任务、读代码、改代码、自己验证、遇到需要我手动操作的（比如去 Supabase SQL Editor 跑 SQL）就明确告诉我。
