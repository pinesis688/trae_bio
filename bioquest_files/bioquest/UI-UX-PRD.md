## 2. 用户画像与场景

### 2.1 核心用户画像

| 画像 | 特征 | 核心需求 |
|------|------|----------|
| **竞赛冲刺者** | 高二/高三，备考全国生物学联赛，每日学习 3-5 小时 | 高频刷题、错题复盘、模考模拟、薄弱点定位 |
| **日常学习者** | 高一/高二学生，按课标进度学习必修/选择性必修 | 知识卡片巩固、生物动画理解过程、章节练习 |
| **好奇探索者** | 对生物感兴趣的初学者或跨学科学生 | 知识图谱浏览、虚拟实验室体验、趣味内容 |
| **教师用户** | 中学生物教师，管理班级学生 | 学情监控、布置任务、资源推荐、答疑 |

### 2.2 核心使用场景

1. **竞赛备考流**：首页 → 模考 → 查看解析 → 错题收录 → 错题复盘 → 针对性练习
2. **日常学习流**：首页 → 知识卡片 → 生物动画理解概念 → 专项练习 → 学习分析
3. **错题修复流**：练习/模考中答错 → 错题本 → AI 分析 → 知识图谱关联 → 推送练习
4. **实验理解流**：学习章节 → 虚拟实验室 → 选择实验 → 步骤操作 → 生成报告
5. **学习管理流**：仪表盘 → 今日推荐 → 番茄钟专注 → 完成待办 → 学习统计

## 7. 组件库规范

### 7.1 按钮 (Button)

| 变体 | 样式 | 使用场景 |
|------|------|----------|
| Primary Hero | `bg: --color-deep`, 文字白色, `shadow-button` | Hero CTA、关键行动 |
| Secondary CTA | `border: 2px solid --color-deep`, 文字深色 | 次要行动 |
| Ghost | 透明背景, `border: 1px solid --border-default` | 取消、返回 |
| Danger | `bg: --color-error`, 文字白色 | 删除、退出 |

**状态**：Default → Hover (亮度+5%) → Active (亮度-5%, scale 0.98) → Disabled (opacity 0.4, cursor not-allowed) → Loading (文字替换为 spinner)

**尺寸**：

| 尺寸 | 高度 | 内边距 | 字号 | 圆角 |
|------|------|--------|------|------|
| SM | 32px | 8px 16px | 0.85rem | 6px |
| MD | 40px | 10px 20px | 0.94rem | 8px |
| LG | 48px | 12px 28px | 1rem | 10px |

### 7.2 卡片 (Card)

| 变体 | 样式 | 使用场景 |
|------|------|----------|
| Content Card | `bg: --surface-primary`, `border: 1px solid --border-light`, `radius: --radius-lg` | 模块选择、错题展示 |
| Stats Card | 同上 + 内部图标 + 数值 + 标签 | 统计数据展示 |
| Action Card | 同上 + 底部 CTA 按钮 | 推荐操作、待办事项 |

**交互**：Hover `translateY(-2px)` + `shadow-md`，Click `scale(0.99)`

### 7.3 输入框 (Input)

样式：`border: 1px solid --border-default`, `radius: --radius-sm`, `padding: 10px 14px`

**状态**：Default → Focus (`border-color: --color-amber`, `box-shadow: 0 0 0 3px rgba(196,149,106,0.15)`) → Error (`border-color: --color-error`) → Disabled (opacity 0.5)

### 7.4 标签 (Tag/Badge)

| 变体 | 样式 |
|------|------|
| Module Tag | `bg: rgba(90,125,92,0.1)`, `color: --color-sage`, `radius: --radius-full` |
| Status Tag | 根据状态使用语义色背景 + 对应文字色 |
| Count Badge | `bg: --color-amber`, `color: white`, 圆形，`font-size: 0.7rem` |

### 7.5 导航 Tab

样式：文字标签，`padding: 8px 16px`，底部 2px 指示条

**状态**：Default (文字 `--text-muted`) → Hover (文字 `--text-secondary`) → Active (文字 `--color-deep`，底部指示条 `--color-amber`)

### 7.6 Toast 通知

位置：右上角固定

样式：`bg: --surface-primary`，`shadow-lg`，`radius: --radius-md`，自动消失（3-5s）

类型：Success (绿色左侧条) / Error (红色左侧条) / Info (蓝色左侧条)

### 7.7 模态框 (Modal)

结构：遮罩层 `rgba(0,0,0,0.5)` + 居中面板

面板样式：`bg: --surface-primary`，`radius: --radius-lg`，`shadow-lg`，`max-width: 480px`

包含：标题 + 内容 + 操作按钮（取消 + 确认）

关闭方式：点击遮罩 / 点击 X / 按 ESC

### 7.8 进度条 (Progress Bar)

样式：圆角轨道 `bg: --surface-tertiary`，填充条 `bg: --color-sage`，支持渐变色（番茄钟可用 amber 渐变）

动画：填充时使用 `--transition-base` 过渡

### 7.9 空状态 (Empty State)

结构：图标（使用 SVG）+ 标题 + 描述文字 + 操作按钮

示例：「还没有错题」+ 「去练习中收集错题吧」+ [开始练习]

### 7.10 加载状态 (Loading)

| 类型 | 样式 | 场景 |
|------|------|------|
| Spinner | CSS 旋转动画，`color: --color-sage` | 按钮加载、小区域 |
| Skeleton | 灰色矩形块，渐变闪烁动画 | 列表加载、卡片加载 |
| Progress | 页面顶部细条（3px），颜色 `--color-amber` | 路由切换 |

## 9. 动效规范

### 9.1 页面过渡

| 场景 | 动效 | 时长 |
|------|------|------|
| 路由切换 | 内容淡入 (opacity 0→1) + 轻微上移 (translateY 8px→0) | 0.3s |
| 首次加载 | Hero 内容分层渐入（label → title → subtitle → CTA，依次 0.1s） | 0.8s 总 |

### 9.2 微交互

| 场景 | 动效 | 约束 |
|------|------|------|
| 按钮点击 | `scale(0.97)` → 弹回 | 150ms |
| 卡片悬浮 | `translateY(-2px)` + 阴影加深 | 200ms ease |
| 复选框选中 | 缩放弹跳 (spring curve) | 300ms |
| 答案正确 | 绿色闪光 + 对勾图标缩放弹出 | 500ms |
| 答案错误 | 红色震动 (translateX ±4px) | 400ms |
| 错题收录 | 卡片滑入侧边列表 | 400ms ease |
| 倒计时数字翻转 | 数字从上方滑入 + 旧数字向下滑出 | 300ms |

### 9.3 Canvas 动画性能约束

| 指标 | 要求 |
|------|------|
| 帧率 | ≥ 60fps（普通笔记本） |
| 首屏渲染 | < 100ms（首帧） |
| 内存占用 | < 50MB（单个 Canvas） |
| 触屏响应 | < 16ms 输入延迟 |
| 暂停/不可见 | `requestAnimationFrame` 自动暂停 |
| 缩放 | 支持 1x-3x DPR |

## 10. 无障碍规范 (WCAG 2.1 AA)

### 10.1 文字对比度

| 场景 | 最低对比度 |
|------|------------|
| 正常文字 (< 18px) | 4.5:1 |
| 大文字 (≥ 18px / 14px bold) | 3:1 |
| UI 组件（按钮边框、图标） | 3:1 |
| 焦点指示器 | 3:1 against adjacent colors |

### 10.2 键盘导航

- 所有可交互元素支持 `Tab` 聚焦
- 聚焦态使用 `box-shadow: 0 0 0 2px rgba(196,149,106,0.6)` 轮廓
- `Enter` / `Space` 激活按钮和链接
- `Escape` 关闭模态框、下拉菜单
- 首页已有 skip-link (`跳到主要内容`)，所有页面均需保留

### 10.3 语义化 HTML

- 使用 `<nav>`, `<main>`, `<section>`, `<header>`, `<footer>` 等语义标签
- 按钮使用 `<button>`，不使用 `<div onclick>`
- 图片提供 `alt` 文本
- 表单元素关联 `<label>`
- 重要通知使用 `aria-live="polite"` / `aria-live="assertive"`

### 10.4 减弱动效

```css
@media (prefers-reduced-motion: reduce) {
  *, *::before, *::after {
    animation-duration: 0.01ms !important;
    animation-iteration-count: 1 !important;
    transition-duration: 0.01ms !important;
  }
}
```

## 12. 导航重构方案

### 12.1 问题分析

当前桌面端导航项 17 个，超出用户短期记忆容量（7±2）。信息过载导致：
- 导航栏水平溢出（小屏幕）
- 用户难以快速定位目标功能
- 功能权重不清晰

### 12.2 分组折叠方案

将 17 项导航重组为 7 个一级入口：

| 一级入口 | 类型 | 内容 |
|----------|------|------|
| 首页 | 直达 | `/` |
| 练习 | 直达 | `/practice` |
| 模考 | 直达 | `/exam` |
| 复习 | 下拉菜单 | 错题本 `/wrongbook`、诊断 `/diagnosis`、图谱 `/knowledge-graph`、卡片 `/cards` |
| 实验室 | 下拉菜单 | 虚拟实验室 `/bio-lab`、生物动画 `/bio-animation`、录题 `/photo-quiz` |
| 社区 | 直达 | `/community`（整合讨论、悬赏） |
| 我的 | 图标+下拉 | 仪表盘 `/dashboard`、学习管理 `/study`、排行榜 `/leaderboard`、教师 `/teacher`、设置 |

### 12.3 下拉菜单交互

- 触发：Hover (桌面) / Click (触屏设备)
- 展开动画：`translateY(-4px) → translateY(0)` + `opacity 0→1`，200ms
- 关闭：鼠标离开菜单区域后 150ms 延迟关闭
- 选中态：当前页面所在的菜单组高亮显示

## 15. 全局交互模式

### 15.1 表单交互

- 实时验证：输入后 500ms 防抖验证（用户名唯一性、邮箱格式等）
- 提交后：按钮变为 Loading 态，成功后 Toast 通知
- 错误：字段下方红字提示 + 输入框红色边框
- 重试：网络错误时显示内联重试按钮

### 15.2 列表交互

- 加载更多：滚动到底部自动触发（Infinite Scroll）或「加载更多」按钮
- 搜索/筛选：输入即时过滤，无需手动提交
- 排序：点击表头切换升/降序，当前排序字段高亮
- 空状态：列表为空时显示引导性空状态组件

### 15.3 权限引导

未登录用户访问需要登录的功能时：
1. 显示功能预览（模糊化或限制部分内容）
2. 底部显示「登录后解锁完整功能」提示
3. 点击跳转到登录页，登录后自动返回目标页

## 16. 设计原则总结

1. **Botanical Laboratory 品牌一致性**：所有页面遵循深绿 + 琥珀金的植物实验室主题，杜绝风格割裂
2. **学习效率优先**：减少认知负荷，核心路径点击 ≤ 3 次到达目标功能
3. **渐进式信息展示**：先概览后细节，可折叠面板默认收起
4. **反馈即时化**：所有用户操作 200ms 内有视觉反馈
5. **容错友好**：错误操作可撤销，不使用惩罚性视觉（保留"不扣 CR"原则）
6. **移动端优先思考**：每个新功能先考虑移动端布局，再扩展桌面端
7. **无障碍内置**：不是附加，而是默认 — 对比度、键盘导航、语义化 HTML
